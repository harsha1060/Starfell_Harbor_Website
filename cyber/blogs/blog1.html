Analyzing the 'Hack The Box' Challenge

By Your Name

Welcome to a detailed walkthrough of one of the most popular machines on Hack The Box. This post will guide you through the entire process, from initial reconnaissance to gaining root access. The goal is not just to find the flags, but to understand the methodology behind each step.

The true art of hacking is not in the tools you use, but in the mindset you cultivate. It's about curiosity, persistence, and seeing the world through a different lens.

Phase 1: Enumeration

The first step in any penetration test is gathering information. We started with a standard Nmap scan to identify open ports and services running on the target machine. The scan revealed several interesting ports, including an open web server on port 80 and an FTP server on port 21 that allowed anonymous login.

nmap -sC -sV -oA nmap/initial 10.10.10.X

Exploring the web server, we found a simple webpage with not much functionality. However, directory busting with Gobuster revealed a hidden /admin directory, which led us to a login panel. This became our primary point of interest.

Phase 2: Exploitation

After failing to guess default credentials on the admin panel, we turned our attention back to the FTP server. We found a configuration file that contained what appeared to be hashed credentials. Using an online hash identifier and John the Ripper, we were able to crack the password for a user named 'admin'.

With these credentials, we successfully logged into the admin panel. The panel allowed file uploads, but with strict filtering for common web shell extensions like .php. By renaming our shell to .phtml, we successfully bypassed the filter and uploaded a reverse shell.

Privilege Escalation

Once we had our initial shell as the www-data user, the next step was to escalate our privileges to root. We ran linpeas.sh to automate the search for potential privilege escalation vectors. The script highlighted a cron job running with root privileges every five minutes. This script was writable by our user.

# Contents of the cron job script
echo "Running cleanup..." > /var/log/cleanup.log
# We can inject our payload here

We modified the script to include a command that would copy the /bin/bash binary to a new file and set the SUID bit. After waiting for the cron job to execute, we had a new bash shell with root permissions, allowing us to capture the final flag.
